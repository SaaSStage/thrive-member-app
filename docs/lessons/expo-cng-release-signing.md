# Release signing & versioning for this app (Expo CNG — android/ is regenerated)

**Context:** `android/` is fully gitignored (`/android` in `.gitignore`; 0 tracked files) — it's
Continuous Native Generation, regenerated by `expo prebuild`. So anything dropped *inside* `android/`
(the keystore, signing edits in `build.gradle`) is **local-only and wiped on a clean prebuild**.

**How signing is wired (so a future build / fresh clone reproduces it):**
- The production **"Thrive Radio" upload key** lives at `credentials/thrive-radio-key.jks`
  (alias `thrive-radio`); store/key passwords in `credentials/key.properties`. **`/credentials/` is
  gitignored** — never commit the keystore or passwords. Keep a secure backup elsewhere; losing this
  key means you can never update the published app again.
- `android/app/build.gradle` has a `release` signingConfig that loads
  `rootProject.file('../credentials/key.properties')`, and the release buildType uses it
  (falls back to debug signing if the file is absent). These edits are **not committed** (android/ is
  ignored) — **re-apply them after any `expo prebuild --clean`** (or promote to a config plugin if it
  recurs).
- **Version source of truth is `app.json`** (committed): `expo.version` + `expo.android.versionCode`.
  build.gradle picks them up on prebuild. Currently `2.6.0` / `25`.

**Build commands** (set `ANDROID_HOME`, run from `android/`):
- APK (sideload/testing): `./gradlew assembleRelease` →
  `android/app/build/outputs/apk/release/app-release.apk`
- AAB (Google Play): `./gradlew bundleRelease` →
  `android/app/build/outputs/bundle/release/app-release.aab`
- Verify signature: `apksigner verify --print-certs <apk>` → SHA-1 must be
  `89:CC:4D:FD:B9:A4:BF:D5:67:80:16:4F:C9:B6:CD:B8:39:85:6A:12` (the production cert).

**Windows gotcha:** `key.properties` `storeFile` must use **forward slashes** (backslashes get
mangled by Java `.properties` escaping). See [[android-keystore-windows-properties-forward-slash]].

**Distribution:** ship as an upgrade of `com.thriveradio.app`, not a new app — see
ADR `docs/decisions/0003-ship-as-play-upgrade-of-thrive-radio.md`.
