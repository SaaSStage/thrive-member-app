## Next step (what needs YOU — physical device + Apple 2FA)
1. **Open the workspace:** `open ios/thrivememberapp.xcworkspace` (use `.xcworkspace`, not the project).
2. **Signing:** select the `thrivememberapp` target → Signing & Capabilities → check **Automatically
   manage signing** → pick your **Team** (sign into Xcode with the Apple ID on the existing
   `com.thriveradio.app` listing; the Team ID isn't stored in the repo, so Xcode supplies it). Bundle
   id must stay `com.thriveradio.app` — it's an in-place upgrade; it cannot change.
3. **Run on device:** plug in the iPhone, trust it, select it as the run destination, ⌘R. Then verify
   the **voice flow**: record a sample and confirm the on-device **quality gate** passes. The gate
   thresholds are **unvalidated on hardware** — tune them here. Key ones (see
   `docs/specs/voice-quality-gate-hardening.md`): SNR ≥ 18 dB; voiced-seconds floors
   (sustained_vowel ≥ 4 s continuous, reading_passage ≥ 20 s total, diadochokinetic ≥ 6 s total);
   peak window −30 to −1 dBFS. Adjust in `DEFAULT_THRESHOLDS` / `RECORDING_CONFIG` if real takes
   trip them.
4. **Confirm before upload (one screen):** bundle id `com.thriveradio.app`, your Team, version
   `2.6.0`, buildNumber `3`, targeting the **existing** App Store Connect listing.
   - ⚠️ **Verify buildNumber 3 is one higher than the last iOS build actually uploaded** to that
     listing (check App Store Connect → TestFlight/Builds). buildNumber 3 is an assumption; iOS
     uses `CFBundleVersion`, independent of Android's versionCode (25/26). If ASC already has a
     build ≥ 3, bump `ios.buildNumber` in app.json and re-run prebuild.
5. **Archive & upload:** Xcode → Product → **Archive** (Release, real device / "Any iOS Device") →
   Distribute App → App Store Connect → Upload. Do **NOT** create a new App Store Connect app.

## Watch out
- **Deprecation (eventual fix):** `VoiceRecorderModule.swift` uses
  `AVAudioSession.sharedInstance().recordPermission`, deprecated on iOS 17+. It compiles (warning
  only) but should migrate to `AVAudioApplication.shared.recordPermission` /
  `AVAudioApplication.requestRecordPermission` before the deprecation becomes an error.
- **CNG:** after any `expo prebuild --clean`, `ios/` is regenerated — never hand-edit native files
  expecting them to survive. The Swift fix is safe because it lives in `modules/voice-recorder`
  (source), not `ios/`.
- **Simulator build only:** compilation is proven, but actual **recording on hardware** and the
  quality-gate thresholds are still unverified — that's step 3 above.
- Reproduce the Simulator build:
  `xcodebuild -workspace ios/thrivememberapp.xcworkspace -scheme thrivememberapp -configuration Debug -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 17' CODE_SIGNING_ALLOWED=NO build`
